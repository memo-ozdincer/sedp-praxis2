Praxis II: Automated LEGO Brick Sorter

Engineering an Accessible and Accurate Sorting Solution for ToroLUG

Project Summary

Addressing the time-consuming task of manual LEGO sorting for the Toronto LEGO User Group (ToroLUG), this Praxis II project involved designing and prototyping an automated system. The primary goal, driven by stakeholder needs (Stakeholder Analysis), was achieving high sorting accuracy (target 99%) for various piece types, including special printed/stickered ones, into user-defined categories.

Our team (Memo Ozdincer, Nancy Fang, Rada Anastanova, Jadon Tsai) followed a structured design process. After decomposing the problem (Problem Decomposition) into functions like feeding, isolation, recognition, and sorting, we used divergent methods (WBN, Personal Analogy, Lotus Blossom) to generate concepts. Evaluation (Pugh Chart) led to a hybrid solution: a dual-stage feed/isolation mechanism, AI-powered computer vision recognition, and a servo-driven rotating sorting platform.

My core contributions focused on the recognition system development (AI model training, vision system integration) and the overall system integration and testing framework. Iterative prototyping (Prototype Testing) was essential, especially for optimizing the feed system's piece isolation (achieving 94% success) which was critical for recognition accuracy. The final integrated prototype achieved 97% overall sorting accuracy at 16 pieces/minute, successfully demonstrating the concept's viability and meeting key user requirements. The project concluded with comprehensive documentation (Technical Documentation) and a user-focused presentation (User-Focused Presentation) to ToroLUG.

1. Stakeholder Engagement & Requirements

Engaging with ToroLUG members revealed that high accuracy (99% target) and handling special pieces were far more critical than raw speed. Customization of sorting categories was also essential.

CTMF Applied: Stakeholder Analysis (#1)

Crucial for understanding the high value placed on accuracy over speed, fundamentally shaping the project's technical direction.

Assessment: Validated the importance of direct user input to avoid designing based on incorrect assumptions.

CTMF Applied: Requirements Development (#2)

Formalized the 99% accuracy target and other key metrics (throughput, categories, no damage) providing clear evaluation criteria.

Assessment: The high accuracy requirement drove the selection of AI-vision and necessitated focus on the feed system's reliability.

CTMF Applied: Problem Decomposition (#3)

System broken into Feed → Isolate → Recognize → Sort → Collect. Highlighted 'Isolate' as critical bottleneck for recognition accuracy.

Assessment: Essential for managing complexity and focusing effort on the critical isolation sub-problem.

2. Concept Generation & Evaluation

Multiple ideation methods generated diverse concepts. WBN explored aspirational goals (like cleaning), Personal Analogy highlighted piece fragility, and Lotus Blossom systematically explored mechanism types.

Pugh Chart analysis compared major concepts (Linear Conveyor, LEGO Shaker, Rotating Plate, Color Sorter UI), strongly favoring the AI/UI approach for accuracy and customization potential.

WBN (#4)

Helped scope project by identifying 'cleaning' as low value/high complexity. Focused efforts on core sorting.

Personal Analogy (#5)

Generated critical 'No Damage' requirement (#R11) by empathizing with the LEGO piece.

Pugh Chart (#7)

Quickly highlighted the potential of the AI/UI concept for meeting accuracy/customization needs, guiding hybrid design.

3. Prototype Development & Testing

Iterative prototyping focused on subsystems: Feed (P1), Recognition (P2), Sorting (P3), then Integration (P4). Proxy testing was key, especially for the feed system where isolation success improved from 68% to 94% through refinement (vibration + shutter).

CTMF Applied: Prototype Testing Framework (#11)

Developed proxy tests (Operational Reliability, UI/UX) with specific metrics (accuracy, cycle time, intuitiveness) linked to requirements. Used standardized brick sets.

Assessment: Critical for identifying the feed system bottleneck and validating improvements. Provided quantitative evidence of meeting performance targets (97% accuracy, 16 pieces/min).

4. Final Solution & Reflection

The final design integrates AI vision with precision mechanical handling, achieving 97% accuracy across customizable categories at 16 pieces/minute. It features a dual-stage isolation feed, AI recognition (type, color, special features), and a rotating sorting platform.

Reflection

This project exemplified my design philosophy's focus on Specificity of Purpose – concentrating on the critical 99% accuracy requirement shaped the entire technical approach. Continuous stakeholder engagement (Stakeholder Analysis) was vital for aligning technical possibilities with real user value. Iterative prototyping (Prototype Testing) and tackling the difficult feed system (Embracing Discomfort) were key to success. My work on the AI recognition system and testing framework directly applied my analytical strengths. Communicating the solution's value to ToroLUG (User-Focused Presentation) required translating technical accuracy into tangible user benefits (more building time).

CTMF Applied: Documentation (#10) & Presentation (#12)

Comprehensive documentation traced design decisions to requirements/testing. Annotated bibliography linked choices to research. Final presentation focused on demonstrating value to ToroLUG, translating accuracy into user benefits.

Assessment: Clear documentation ensured rigor and team alignment. User-focused presentation was crucial for stakeholder buy-in and demonstrating project success beyond technical metrics.
